W2GO - API launch

Backend to a React App that generates itineraries based on user input

Rest API and intermediary between the frontend application and Google Maps Places API. It handles GET requests to retrieve a list of places nearby the user's location and GET, PUT, POST and DELETE requests for custom "Step" and "Itinerary" objects which are stored in a database. It's already up and running but we are facing some challenges, such as:

note_add

Reducing response time for a large list of places

 note_add

Reducing the number of API calls to Google Maps

 note_add

Better dynamic instantiation of compute engines to support large number of requests

Current tech stack was chosen for ease of use and rapid implementation, but given the challenges we are facing we are considering migrating the APIs resources towards independent, robust and reliable microservices.
